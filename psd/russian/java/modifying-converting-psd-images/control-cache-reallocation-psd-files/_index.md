# Управление перераспределением кэша в PSD‑файлах

## Введение
Если вам нужно эффективно **create PSD image java** проекты, контроль перераспределения кэша имеет решающее значение. При программной работе с изображениями и файлами Photoshop эффективность является ключевым фактором. Aspose.PSD for Java предлагает мощные возможности для управления и манипулирования PSD‑файлами без проблем. Одним из фундаментальных аспектов оптимизации производительности является контроль перераспределения кэша. Управление кэшем помогает поддерживать баланс между использованием памяти и диска, обеспечивая плавную работу вашего приложения без неожиданных сбоев или замедлений. 

## Быстрые ответы
- **Что делает перераспределение кэша?** Оно балансирует использование памяти и диска при обработке больших PSD‑файлов.  
- **Какой тип кэша лучше всего подходит для больших изображений?** `CacheOnDiskOnly` освобождает память, сохраняя кэш на диске.  
- **Сколько дискового пространства я могу выделить?** До 1 ГБ (или любого размера, который вы зададите с помощью `setMaxDiskSpaceForCache`).  
- **Нужна ли лицензия для использования этих функций?** Лицензия снимает ограничения пробной версии; см. страницу покупки Aspose.  
- **Можно ли мониторить использование кэша во время выполнения?** Да, используйте `Cache.getAllocatedDiskBytesCount()` и `Cache.getAllocatedMemoryBytesCount()`.

## Почему важно контролировать перераспределение кэша?
Управление кэшем критично, когда вы **create PSD image java** приложения, работающие с изображениями высокого разрешения или многослойными файлами. Правильные настройки кэша предотвращают ошибки «недостаточно памяти», снижают паузы сборщика мусора и обеспечивают предсказуемую производительность на серверах или настольных приложениях.

## Как создать PSD Image Java с управлением кэшем
Ниже представлена пошаговая инструкция, связывающая настройку кэша непосредственно с процессом создания PSD‑изображения в Java.

### Шаг 1: Настройка каталога данных
Сначала вам нужно создать каталог, в котором будут храниться файлы кэша. Это необходимо для эффективного управления кэшем.
```java
String dataDir = "Your Document Directory";
Cache.setCacheFolder(dataDir);
```

- `String dataDir`: Определяет каталог для кэша вашего документа.  
- `Cache.setCacheFolder(dataDir)`: Этот метод задаёт папку кэша в указанный каталог. Любой кэш, созданный Aspose, теперь будет сохраняться здесь вместо стандартного временного каталога.

### Шаг 2: Настройка кэша на диск
Далее укажем, что кэш должен храниться только на диске. Это особенно полезно, если ваше приложение работает с большими файлами и вы хотите освободить память.
```java
Cache.setCacheType(CacheType.CacheOnDiskOnly);
```

- `Cache.setCacheType(CacheType.CacheOnDiskOnly)`: Эта опция гарантирует, что кэш не будет удерживаться в памяти, что полезно при работе с большими PSD‑файлами без избыточного потребления ОЗУ.

### Шаг 3: Установка максимального размера кэша на диске и в памяти
Теперь ограничим размеры кэша. Это важно, поскольку неограниченный кэш может привести к проблемам с производительностью.
```java
Cache.setMaxDiskSpaceForCache(1073741824); // 1 gigabyte
Cache.setMaxMemoryForCache(1073741824); // 1 gigabyte
```

- `Cache.setMaxDiskSpaceForCache(1073741824)`: Устанавливает ограничение в 1 ГБ для кэша на диске. Вы можете изменить этот размер в зависимости от ваших потребностей.  
- `Cache.setMaxMemoryForCache(1073741824)`: Аналогично, ограничивает кэш в памяти, гарантируя, что приложение не будет использовать избыточную память.

### Шаг 4: Управление стратегией перераспределения кэша
Управление тем, как кэш перераспределяется, необходимо для поддержания производительности. Вот как это настроить.
```java
Cache.setExactReallocateOnly(false);
```

- `Cache.setExactReallocateOnly(false)`: При значении `false` Aspose может более гибко управлять перераспределением кэша, что может привести к лучшей производительности.

### Шаг 5: Проверка выделенного размера кэша
Этот шаг посвящён мониторингу количества байт, выделенных в данный момент для кэша в памяти или на диске. Реализуем это:
```java
long l1 = Cache.getAllocatedDiskBytesCount();
long l2 = Cache.getAllocatedMemoryBytesCount();
```

- `long l1`: Сохраняет количество байт, выделенных на диске.  
- `long l2`: Сохраняет количество байт, выделенных в памяти.  
Вы можете проверять эти значения в любой момент, чтобы убедиться, что приложение управляет памятью и дисковым пространством должным образом.

### Шаг 6: Создание PSD‑изображения
Теперь, когда настройки кэша заданы, создадим простое PSD‑изображение.
```java
PsdOptions options = new PsdOptions();
Color[] color = { Color.getRed(), Color.getBlue(), Color.getBlack(), Color.getWhite() };
options.setPalette(new ColorPalette(color));
options.setSource(new StreamSource(new ByteArrayInputStream(new byte[0])));
RasterImage image = (RasterImage) Image.create(options, 100, 100);
```

- `PsdOptions options`: Этот объект позволяет задавать параметры при создании изображения Photoshop.  
- `Color[] color`: Массив, содержащий цвета, которые будут использованы в палитре изображения.

### Шаг 7: Сохранение пиксельных данных в изображение
Заполним изображение пиксельными данными и сохраним его.
```java
Color[] pixels = new Color[10000];
for (int i = 0; i < pixels.length; i++) {
    pixels[i] = Color.getWhite();
}
image.savePixels(image.getBounds(), pixels);
```

- `Color[] pixels`: Это массив объектов цвета. Здесь мы заполняем его белыми пикселями.  
- `image.savePixels(image.getBounds(), pixels)`: Этот метод сохраняет пиксельные данные в изображение. Он обновляет изображение заданными нами цветами.

### Шаг 8: Мониторинг выделенного кэша после создания изображения
После создания изображения рекомендуется снова проверить, сколько байт выделено для кэша.
```java
long diskBytes = Cache.getAllocatedDiskBytesCount();
long memoryBytes = Cache.getAllocatedMemoryBytesCount();
```

- `long diskBytes`: Захватывает текущее выделение на диске после создания изображения.  
- `long memoryBytes`: Захватывает текущее выделение в памяти.  
Этот шаг поможет определить, сколько кэша потребляется после ваших операций.

### Шаг 9: Проверка правильного освобождения
Наконец, важно убедиться, что все объекты Aspose.PSD были корректно освобождены, чтобы избежать утечек памяти.
```java
l1 = Cache.getAllocatedDiskBytesCount();
l2 = Cache.getAllocatedMemoryBytesCount();
```

- `l1` и `l2`: Эти переменные помогут проверить окончательное выделение. Если они не равны нулю, это указывает на то, что некоторые объекты не были правильно освобождены.

## Распространённые проблемы и решения
- **Cache folder not created** – Убедитесь, что приложение имеет права записи для пути, переданного в `Cache.setCacheFolder`.  
- **Out‑of‑memory errors** – Проверьте, что `Cache.setCacheType(CacheType.CacheOnDiskOnly)` применён до загрузки больших PSD‑файлов.  
- **Unexpected cache size** – Используйте методы `Cache.getAllocated*BytesCount()` после каждой крупной операции, чтобы отслеживать рост.
