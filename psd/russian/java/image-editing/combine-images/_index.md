# Создание PSD‑файла в Java – Объединение изображений с помощью Aspose.PSD

## Введение

Если вам нужно **создать PSD‑файл в Java**, объединив несколько картинок, Aspose.PSD делает эту задачу простой. В этом руководстве мы пройдем процесс объединения двух изображений в один холст PSD, объясним, почему такой подход полезен, и предоставим готовый к запуску код. К концу вы сможете **combine two images Java** и сгенерировать профессиональный многослойный файл.

## Быстрые ответы
- **Какая библиотека лучше всего подходит для объединения изображений в PSD?** Aspose.PSD for Java.  
- **Сколько времени занимает реализация?** Около 10‑15 минут для базового объединения.  
- **Нужна ли лицензия?** Бесплатная пробная версия подходит для тестирования; для продакшна требуется коммерческая лицензия.  
- **Можно ли добавить более двух изображений?** Да – повторите вызовы `drawImage` для каждого дополнительного слоя.  
- **Какая версия Java поддерживается?** Java 8 и новее.

## Требования

Перед тем как приступить, убедитесь, что у вас есть следующее:

1. **Aspose.PSD Library** – скачайте её [здесь](https://releases.aspose.com/psd/java/).  
2. **Java Development Kit (JDK)** – рекомендуется Java 8+.  
3. **Document Directory** – папка на вашем компьютере, где будут храниться исходные изображения и полученный PSD.

## Импорт пакетов

Начните с импорта необходимых классов Aspose.PSD в ваш проект:

```java
import com.aspose.psd.Color;
import com.aspose.psd.Graphics;
import com.aspose.psd.Image;

import com.aspose.psd.imageoptions.PsdOptions;
import com.aspose.psd.sources.FileCreateSource;
```

## Пошаговое руководство

### Шаг 1: Создание PSD‑Options (подготовка файла)

Сначала создаём экземпляр `PsdOptions`, который будет хранить настройки вывода.

```java
PsdOptions imageOptions = new PsdOptions();
```

### Шаг 2: Установка FileCreateSource (определение места сохранения PSD)

Назначаем `FileCreateSource` параметрам, указывая путь к желаемому результирующему файлу.

```java
imageOptions.setSource(new FileCreateSource(dataDir + "Two_images_result_out.psd", false));
```

### Шаг 3: Создание экземпляра Image (инициализация размера холста)

Создаём объект `Image` с указанными параметрами и задаём холст размером 600 × 600 пикселей.

```java
Image image = Image.create(imageOptions, 600, 600);
```

### Шаг 4: Инициализация Graphics и отрисовка первого изображения

Объект `Graphics` позволяет рисовать на холсте. Очищаем фон до белого и рисуем первое исходное изображение на левой половине.

```java
Graphics graphics = new Graphics(image);
graphics.clear(Color.getWhite());
graphics.drawImage(Image.load(dataDir + "example1.psd"), 0, 0, 300, 600);
```

### Шаг 5: Отрисовка второго изображения (завершение объединения)

Теперь размещаем второе изображение на правой половине того же холста.

```java
graphics.drawImage(Image.load(dataDir + "example2.psd"), 300, 0, 300, 600);
```

### Шаг 6: Сохранение полученного PSD‑файла

Наконец, сохраняем объединённый холст на диск.

```java
image.save();
```

Поздравляем! Вы успешно **merge images into PSD** и создали PSD‑файл в Java.

## Почему объединять изображения с помощью Aspose.PSD?

- **Layer‑aware** – каждый вызов `drawImage` добавляет отдельный слой, который можно редактировать позже.  
- **Format flexibility** – поддерживает PSD, PNG, JPEG, BMP и многое другое.  
- **No native dependencies** – чистый Java, работает на любой ОС, где запущен JDK.  

## Распространённые проблемы и решения

| Проблема | Причина | Решение |
|----------|---------|---------|
| `File not found` ошибка при загрузке исходных изображений | Неправильный путь `dataDir` | Убедитесь, что `dataDir` указывает на папку, содержащую `example1.psd` и `example2.psd`. |
| Выходной PSD пустой | `graphics.clear` вызывается после отрисовки | Убедитесь, что `graphics.clear(Color.getWhite())` выполняется **до** любых вызовов `drawImage`. |
| Исключение лицензии во время выполнения | Отсутствующая или просроченная лицензия Aspose.PSD | Примените действующую лицензию с помощью `License license = new License(); license.setLicense("Aspose.PSD.lic");` перед любыми вызовами API. |
