# Сохранить PSD с шрифтами и принудительно обновить кэш шрифтов – Aspose.PSD Java

## Введение

Если вам нужно **save PSD with fonts** быстро, одновременно убедившись, что нужные шрифты доступны, вы попали в нужное место. Кеширование шрифтов может значительно **improve image performance**, особенно при повторной обработке больших документов Photoshop. В этом руководстве мы пройдем все шаги по принудительному обновлению кэша шрифтов, **load PSD image** файлов и, наконец, **save PSD with fonts** с использованием Aspose.PSD для Java.

## Краткие ответы
- **Что делает принудительное обновление кэша шрифтов?** Это заставляет Aspose.PSD повторно сканировать системные шрифты, чтобы недавно установленные шрифты сразу же распознавались.  
- **Сколько времени следует ждать установки шрифта?** Пример кода делает паузу на **2 минуты**, что обычно достаточно для ручной установки.  
- **Можно ли использовать это с любым файлом PSD?** Да — при условии, что файл доступен и необходимые шрифты установлены в системе.  
- **Нужна ли лицензия для выполнения этого кода?** Для использования в продакшене требуется временная или полная лицензия; бесплатная пробная версия подходит для оценки.  
- **Какие версии Java поддерживаются?** Aspose.PSD for Java работает с JDK 8 и новее.

## Что такое “save PSD with fonts” и почему это важно?

Сохранение PSD‑файла после изменения его слоёв, текста или шрифтов является последним шагом, фиксирующим ваши изменения. Когда кэш шрифтов не актуален, сохранённый файл может переключиться на шрифты по умолчанию, вызывая визуальные несоответствия. Принудительно обновив кэш сначала, вы гарантируете, что операция **save PSD with fonts** встраивает правильные шрифты.

## Зачем принудительно обновлять кэш шрифтов с помощью Aspose.PSD?

- **Увеличение производительности** — Снижает необходимость повторных поисков шрифтов.  
- **Надёжность** — Гарантирует, что сохранённые PSD‑файлы отображаются точно так, как задумано, на любой машине.  
- **Простота** — Один вызов метода (`OpenTypeFontsCache.updateCache()`) выполняет всю тяжёлую работу.

## Требования

Прежде чем начать, убедитесь, что у вас есть:

- Установленный Java Development Kit (JDK) 8 или новее.  
- Библиотека Aspose.PSD for Java (скачайте с официальной [страницы загрузки](https://releases.aspose.com/psd/java/)).  
- Пример PSD‑файла (`sample.psd`), размещённый в папке, к которой вы можете обратиться из кода.  

Теперь, когда всё готово, давайте перейдём к реализации.

## Импорт пакетов

Сначала импортируйте классы, необходимые для работы с PSD‑изображениями и кэшем шрифтов. Поместите эти инструкции в начале вашего Java‑класса:

```java
import com.aspose.psd.Image;
import com.aspose.psd.OpenTypeFontsCache;

import com.aspose.psd.fileformats.psd.PsdImage;
import java.io.Console;
import java.util.concurrent.TimeUnit;
```

## Пошаговое руководство

### Шаг 1: Загрузка PSD‑изображения (Как загрузить PSD‑изображение)

Мы начинаем с загрузки оригинального PSD‑файла. Это даёт нам базовый объект изображения, который мы позже сможем сохранить заново после обновления кэша шрифтов.

```java
String dataDir = "Your Document Directory";

PsdImage image = (PsdImage)Image.load(dataDir + "sample.psd");
image.save(dataDir + "NoFont.psd");
```

- **Объяснение:**  
  - `Image.load` читает файл в память.  
  - `image.save` создаёт копию с именем **NoFont.psd**, отражающую состояние **before** появления новых шрифтов.  
  - Этот шаг полезен для сравнения и гарантирует, что последующее обновление кэша действительно изменит результат.

### Шаг 2: Ожидание установки шрифта (java thread sleep – improve image performance)

Теперь мы даём пользователю время вручную установить требуемый шрифт. В реальном сценарии вы можете автоматизировать этот шаг, но пауза демонстрирует механизм обновления кэша.

```java
System.out.println("You have 2 minutes to install the font");
Thread.sleep(2 * 60 * 1000);
OpenTypeFontsCache.updateCache();
```

- **Объяснение:**  
  - `Thread.sleep` (это **java thread sleep**) приостанавливает выполнение на **2 минуты** (2 × 60 × 1000 мс).  
  - `OpenTypeFontsCache.updateCache()` заставляет Aspose.PSD повторно сканировать системные OpenType‑шрифты, делая любые недавно установленные шрифты сразу доступными для рендеринга.  
  - Эта пауза помогает **improve image performance**, обеспечивая актуальность кэша перед следующей загрузкой.

### Шаг 3: Загрузка обновлённого PSD‑изображения и **save PSD with fonts**

После обновления кэша мы снова загружаем оригинальный PSD. На этот раз информация о шрифтах актуальна, и мы можем корректно **save PSD with fonts**.

```java
PsdImage image1 = (PsdImage)Image.load(dataDir + "sample.psd");
image1.save(dataDir + "HasFont.psd");
```

- **Объяснение:**  
  - Второй загрузка подхватывает недавно установленный шрифт.  
  - Сохранение в **HasFont.psd** создаёт файл, содержащий правильный рендеринг шрифтов, подтверждая, что обновление кэша сработало.

## Распространённые проблемы и решения

| Проблема | Причина | Решение |
|----------|----------|----------|
| Сохранённый PSD всё ещё показывает шрифт по умолчанию | Шрифт не установлен в месте, сканируемом ОС | Установите шрифт в системную папку шрифтов (например, `C:\Windows\Fonts` в Windows) и повторно запустите `updateCache()`. |
| `Thread.sleep` бросает `InterruptedException` | Сигнатура метода не обрабатывает проверяемые исключения | Добавьте `throws InterruptedException` к вашему `main`‑методу или оберните вызов в блок try‑catch. |
| `OpenTypeFontsCache.updateCache()` ничего не делает | Запуск на безголовом сервере без конфигурации шрифтов | Убедитесь, что на сервере установлены необходимые шрифты и процесс Java имеет права их читать. |
| **handle missing fonts** – PSD рендерится с заменой | Файлы шрифтов повреждены или недоступны | Проверьте целостность файлов шрифтов и убедитесь, что процесс Java имеет доступ для чтения. |
