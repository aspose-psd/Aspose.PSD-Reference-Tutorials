# كيفية إنشاء صورة منحنى بيزيير في جافا

## المقدمة
إذا كنت بحاجة إلى **إنشاء صورة منحنى بيزيير** لتطبيق سطح مكتب جافا أو تطبيق خادم، فإن Aspose.PSD for Java يجعل العملية سهلة بلا عناء. في هذا الدرس سنستعرض رسم منحنى بيزيير ناعم، تخصيص عرض القلم، وحفظ النتيجة كصورة نقطية. في النهاية ستكون مرتاحًا مع طريقة `drawBezier()` ومستعدًا لدمج الرسومات بنمط المتجهات في أي مشروع جافا.

## الإجابات السريعة
- **ما المكتبة الأفضل لرسم المنحنيات في جافا؟** Aspose.PSD for Java توفر واجهة برمجة تطبيقات رسومية متكاملة.  
- **كم من الوقت يستغرق إنشاء صورة منحنى بيزيير أساسية؟** عادةً أقل من 10 دقائق بمجرد إعداد SDK.  
- **ما صيغ الصور المدعومة للتصدير؟** BMP، PNG، JPEG، TIFF، وأكثر.  
- **هل يمكنني تغيير عرض القلم للمنحنى؟** نعم – يسمح لك مُنشئ `Pen` بتحديد أي عرض تحتاجه.  
- **هل أحتاج إلى ترخيص للاستخدام في الإنتاج؟** الترخيص التجاري مطلوب للنشر غير التجريبي.

## ما هو “إنشاء صورة منحنى بيزيير”؟
إنشاء صورة منحنى بيزيير يعني توليد صورة نقطية تحتوي على منحنى معرف رياضيًا. يُعرّف المنحنى بنقطة بداية، نقطتي تحكم، ونقطة نهاية، مما يتيح لك إنتاج أشكال ناعمة وقابلة للتكبير تبدو رائعة بأي حجم.

## لماذا نستخدم Aspose.PSD for Java؟
- **كائنات رسومية غنية** – رسم خطوط، أشكال، ومنحنيات دون الحاجة للتعامل مع تعديل البكسلات منخفض المستوى.  
- **متعدد المنصات** – يعمل على أي نظام تشغيل يدعم جافا.  
- **دعم الدقة العالية** – معالجة قماش كبير دون استهلاك مفرط للذاكرة.  
- **تصدير سهل** – التحويل بين BMP، PNG، JPEG، TIFF بسطر واحد من الشيفرة.

## المتطلبات المسبقة
قبل البدء، تأكد من وجود ما يلي:

1. **مجموعة تطوير جافا (JDK)** – أي نسخة حديثة (8 أو أحدث).  
2. **Aspose.PSD for Java JAR** – قم بتنزيله من [هنا](https://releases.aspose.com/psd/java/) وأضفه إلى مسار الفئات في مشروعك.  
3. **بيئة تطوير متكاملة (IDE)** – Eclipse، IntelliJ IDEA، أو أي محرر تفضله، مُعدّ مع JDK.

## استيراد الحزم
قبل الغوص في التنفيذ، استورد الفئات الضرورية من Aspose.PSD:

```java
import com.aspose.psd.Color;
import com.aspose.psd.Graphics;
import com.aspose.psd.Image;
import com.aspose.psd.Pen;
import com.aspose.psd.fileformats.psd.PsdImage;
import com.aspose.psd.imageoptions.BmpOptions;
```

## دليل خطوة بخطوة

### الخطوة 1: إنشاء كائن صورة
أولًا، تحتاج إلى إنشاء كائن من فئة `PsdImage`، التي تمثل صورة PSD في الذاكرة.

```java
String dataDir = "Your Document Directory";
Image image = new PsdImage(100, 100);
```

*شرح*: يتم إنشاء `PsdImage` بعرض وارتفاع **100 × 100 بكسل** – يمكنك زيادة هذه القيم للحصول على مخرجات بدقة أعلى.

### الخطوة 2: تهيئة سياق الرسومات
بعد ذلك، أنشئ كائنًا من فئة `Graphics` لتنفيذ عمليات الرسم على الصورة.

```java
Graphics graphics = new Graphics(image);
```

*شرح*: يربط كائن `Graphics` أوامر الرسم بـ `image` التي أنشأناها للتو.

### الخطوة 3: مسح سطح الرسومات
امسح سطح الرسومات باستخدام لون خلفية محدد، هنا `Color.getYellow()`.

```java
graphics.clear(Color.getYellow());
```

*شرح*: يضبط خلفية صفراء زاهية لتبرز منحنى البيزيير الأسود.

### الخطوة 4: تهيئة القلم للرسم
قم بإعداد كائن `Pen` بخصائص مثل اللون والعرض لتحديد طريقة رسم المنحنى.

```java
Pen blackPen = new Pen(Color.getBlack(), 3);
```

*شرح*: القلم أسود و**عرضه 3 بكسل** – يمكنك تعديل العرض لجعل المنحنى أسمك أو أرفع (`java graphics pen width`).

### الخطوة 5: تحديد معلمات منحنى البيزيير
حدد نقاط التحكم والنقاط النهائية لمنحنى البيزيير.

```java
float startX = 10, startY = 25;
float controlX1 = 20, controlY1 = 5;
float controlX2 = 55, controlY2 = 10;
float endX = 90, endY = 25;
```

*شرح*:  
- `startX`, `startY` – نقطة بداية المنحنى.  
- `controlX1`, `controlY1` – نقطة التحكم الأولى.  
- `controlX2`, `controlY2` – نقطة التحكم الثانية.  
- `endX`, `endY` – نقطة نهاية المنحنى.

### الخطوة 6: رسم منحنى البيزيير
استخدم طريقة `drawBezier()` لرسم منحنى البيزيير على الصورة باستخدام القلم ونقاط التحكم المحددة مسبقًا.

```java
graphics.drawBezier(blackPen, startX, startY, controlX1, controlY1, controlX2, controlY2, endX, endY);
```

*شرح*: هذه الدعوة الواحدة تُنشئ خطًا ناعمًا **draw bezier curve java** يتبع نقاط التحكم التي زودتها.

### الخطوة 7: حفظ الصورة
احفظ الصورة المرسومة بصيغة BMP.

```java
String outpath = dataDir + "Bezier.bmp";
BmpOptions saveOptions = new BmpOptions();
image.save(outpath, saveOptions);
```

*شرح*: كائن `BmpOptions` يخبر Aspose.PSD بكتابة الملف كملف BMP. يمكنك استبداله بـ `PngOptions` أو `JpegOptions` إذا احتجت صيغة مختلفة.

## المشكلات الشائعة والحلول
- **المنحنى يبدو مسطحًا** – تحقق من إحداثيات نقاط التحكم؛ يجب ألا تكون على خط مستقيم مع نقاط البداية/النهاية.  
- **الصورة فارغة** – تأكد من استدعاء `graphics.clear()` قبل الرسم، وأن لون القلم يتباين مع الخلفية.  
- **OutOfMemoryError على القماش الكبير** – زد حجم ذاكرة JVM (`-Xmx`) أو استخدم صورًا مقسمة إلى مربعات.

## الأسئلة المتكررة
### هل يمكنني رسم عدة منحنيات بيزيير في نفس الصورة؟
نعم، يمكنك رسم عدة منحنيات بتكرار العملية داخل حلقة، مع تغيير نقاط التحكم والنقاط النهائية حسب الحاجة.

### كيف يمكنني تغيير لون منحنى البيزيير؟
غيّر خاصية اللون لكائن `Pen` (`Color.getBlack()` في المثال) قبل استدعاء `drawBezier()`.

### هل Aspose.PSD for Java مناسب للصور عالية الدقة؟
نعم، يدعم Aspose.PSD for Java الصور عالية الدقة مع إدارة فعّالة للذاكرة.

### هل يمكنني تصدير الصورة إلى صيغ غير BMP؟
نعم، يدعم Aspose.PSD for Java تصدير الصور إلى صيغ متعددة مثل PNG، JPEG، TIFF، وغيرها.

### أين يمكنني العثور على المزيد من الأمثلة والوثائق؟
قم بزيارة [توثيق Aspose.PSD لجافا](https://reference.aspose.com/psd/java/) للحصول على أدلة شاملة وعينات شفرة.
