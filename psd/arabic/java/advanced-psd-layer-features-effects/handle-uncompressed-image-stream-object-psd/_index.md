# تصدير PSD إلى PNG – إنشاء كائن رسومات PSD – تدفق غير مضغوط في Java

## المقدمة
مرحبًا بك في عالم معالجة الصور في Java! في هذا الدرس ستقوم **بإنشاء كائن رسومات PSD**، وتعامل مع كائنات تدفق الصور غير المضغوطة، وتتعلم كيفية **تصدير PSD إلى PNG** باستخدام Aspose.PSD for Java. سواء كنت مصممًا جرافيكيًا يسعى لأتمتة سير العمل أو مطور برامج يرغب في دمج قدرات معالجة الصور القوية في تطبيقاته، فإن هذا الدليل مُصمم خصيصًا لك. سنستعرض كل شيء من المتطلبات المسبقة إلى التصدير النهائي، لضمان فهمك الكامل للعملية بأكملها.

## إجابات سريعة
- **ماذا يعني “إنشاء كائن رسومات PSD”؟** يشير إلى إنشاء سياق رسومي لملف PSD بحيث يمكنك الرسم أو تعديل محتوياته.  
- **أي مكتبة تتعامل مع التدفقات غير المضغوطة؟** Aspose.PSD for Java توفر دعمًا كاملاً للبيانات الخام (غير المضغوطة).  
- **هل يمكنني تصدير PSD إلى PNG بعد التعديل؟** نعم—بمجرد حصولك على كائن `Graphics` يمكنك عرض PSD وحفظه كملف PNG.  
- **هل أحتاج إلى ترخيص للتطوير؟** النسخة التجريبية المجانية تكفي للاختبار؛ الترخيص التجاري مطلوب للإنتاج.  
- **هل التصدير بدون فقدان؟** تصدير إلى PNG يحافظ على جودة الصورة، بينما يكون حجم الملف أكبر من JPEG لكنه أصغر من PSD غير المضغوط.  

## كيفية تصدير PSD إلى PNG باستخدام Aspose.PSD for Java
عند الحاجة إلى **تصدير PSD إلى PNG**، يكون سير العمل النموذجي كالتالي:

1. تحميل ملف PSD (أو إنشاؤه).  
2. تنفيذ أي رسم أو تعديل طبقات باستخدام كائن `Graphics`.  
3. حفظ الصورة الناتجة باستخدام `PngOptions` (يمكن إعادة استخدام نفس كائن `Graphics`).  

على الرغم من أن هذا الدرس يركز على التعامل مع التدفقات غير المضغوطة، إلا أن كائن `Graphics` الذي تنشئه يمكن إعادة استخدامه لاحقًا لعرض PSD في ملف PNG ضمن خط أنابيبك.

## استيراد الحزم
لبدء العمل، نحتاج إلى استيراد الحزم الضرورية للعمل مع Aspose.PSD. أدناه ستجد الاستيرادات التي ستحتاجها عادةً لمعالجة ملفات PSD.

```java
import com.aspose.psd.Graphics;
import com.aspose.psd.Image;
import com.aspose.psd.fileformats.psd.CompressionMethod;
import com.aspose.psd.fileformats.psd.PsdImage;
import com.aspose.psd.imageoptions.PsdOptions;
import java.io.ByteArrayInputStream;
import java.io.ByteArrayOutputStream;
```

الآن، لنقسم الشيفرة إلى خطوات قابلة للهضم لضمان قدرتك على المتابعة بسهولة. سنقوم بالإعداد، تحميل ملف PSD، تعديله، وحفظ النتيجة.

## الخطوة 1: تعريف دليل المستندات الخاص بك
قبل بدء كتابة الشيفرة، ستحتاج إلى تحديد مكان وجود ملف PSD الخاص بك. هذا هو بمثابة إعداد المشهد لمشروعك.

```java
String dataDir = "Your Document Directory";
```

استبدل `"Your Document Directory"` بالمسار الفعلي حيث يقع ملف PSD (مثلاً layers.psd). سيساعدك ذلك على العثور على ملفاتك دون عناء.

## الخطوة 2: إنشاء تدفق بايتات للخرج (Byte Array Output Stream)
تحتاج إلى مكان لتخزين الصورة المعدلة قبل القيام بأي شيء بها. سيساعدك `ByteArrayOutputStream` على التقاط بيانات الصورة بسهولة.

```java
ByteArrayOutputStream ms = new ByteArrayOutputStream();
```

هذا السطر يهيئ كائن `ByteArrayOutputStream` جديد يُدعى `ms`. ستستخدم هذا الكائن لحفظ صورتك غير المضغوطة.

## الخطوة 3: تحميل ملف PSD
حان الآن وقت تحميل ملف PSD الفعلي. هنا يبدأ السحر!

```java
PsdImage psdImage = (PsdImage) Image.load(dataDir + "layers.psd");
```

هذا السطر يحمل ملف PSD الخاص بك إلى كائن `PsdImage`. تأكد من صحة المسار؛ وإلا سيظهر خطأ كما لو كان اختبارًا غير مُراقب.

## الخطوة 4: إعداد PsdOptions للحفظ
تحتاج إلى تحديد طريقة حفظ الصورة — غير مضغوطة، بالطبع!

```java
PsdOptions saveOptions = new PsdOptions();
saveOptions.setCompressionMethod(CompressionMethod.Raw);
```

هنا، تنشئ كائن `PsdOptions` وتحدد طريقة الضغط إلى `Raw`. تضمن هذه الطريقة أن تحتفظ الصورة بجودتها الكاملة وتُحفظ دون أي ضغط.

## الخطوة 5: حفظ الصورة إلى تدفق الخرج
```java
psdImage.save(ms, saveOptions);
```

هذا السطر يحفظ الصورة المعدلة في `ByteArrayOutputStream` الذي أنشأته في الخطوة 2، باستخدام الخيارات المحددة في الخطوة 4. تتولى طريقة `save` تشفير الصورة بشكل صحيح بناءً على إعداداتك.

## الخطوة 6: إعادة ضبط تدفق الخرج
بعد الحفظ، يكون تدفق الخرج في نهايته. تحتاج إلى إعادة ضبطه للقراءة من البداية.

```java
ms.reset();
```

طريقة `reset` هذه تُعدّ `ByteArrayOutputStream` للقراءة من البداية مرة أخرى. فكر فيها كإعادة لف شريط قبل الاستماع إلى أغنيتك المفضلة!

## الخطوة 7: تحميل الصورة التي تم إنشاؤها حديثًا
```java
PsdImage img = (PsdImage) Image.load(new ByteArrayInputStream(ms.toByteArray()));
```

هنا، نعيد تحميل الصورة من `ByteArrayOutputStream` إلى كائن `PsdImage` جديد. يمكنك الآن التحقق من نتائج عملك السابق.

## الخطوة 8: إنشاء كائن رسومات (Graphics Object)
للمزيد من التعديل أو العرض، ستحتاج إلى إنشاء كائن رسومات.

```java
Graphics graphics = new Graphics(psdImage);
```

هذا السطر يهيئ كائن `Graphics` باستخدام `psdImage` الخاص بك. الآن يمكنك استخدام هذا الكائن للرسم أو تعديل الصورة حسب الحاجة. إنه كأنك تحمل فرشاة رسم في يدك!

## تعديل طبقات PSD باستخدام كائن الرسومات
الآن بعد أن لديك نسخة **Graphics**، يمكنك **تعديل طبقات PSD**—مثل رسم أشكال، إضافة نص، أو تطبيق فلاتر على طبقة معينة. يعمل سياق الرسومات مباشرةً على بيانات البكسل الأساسية، مما يمنحك تحكمًا دقيقًا في مظهر كل طبقة.

## المشكلات الشائعة والحلول
- **NullPointerException عند تحميل الملف** – تحقق مرة أخرى من مسار `dataDir` وتأكد من صحة اسم الملف.  
- **الخرج مضغوط رغم استخدام Raw** – تأكد من استدعاء `saveOptions.setCompressionMethod(CompressionMethod.Raw);` قبل طريقة `save`.  
- **كائن Graphics يظهر فارغًا** – تأكد من أنك ترسم على نسخة `PsdImage` الصحيحة (استخدم تلك التي قمت بتحميلها، وليس النسخة الجديدة ما لم يكن ذلك مقصودًا).
