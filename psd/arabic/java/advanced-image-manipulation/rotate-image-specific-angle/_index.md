# كيفية تدوير الصورة بزاوية محددة باستخدام Aspose.PSD للـ Java

## المقدمة

إذا كنت بحاجة إلى **كيفية تدوير الصورة** برمجيًا في تطبيق Java، فإن Aspose.PSD للـ Java يقدم واجهة برمجة تطبيقات نظيفة وعالية الأداء تتولى عنك الجزء الثقيل. سواءً كنت تبني محرر صور، أو تولد صورًا مصغرة، أو تُعدّ أصولًا لخدمة ويب، فإن تدوير الصورة بدرجة دقيقة هو طلب شائع. في هذا الدرس سنستعرض العملية بالكامل — من تحميل ملف PSD إلى حفظ النتيجة المدورة — مع إبراز أفضل الممارسات مثل التخزين المؤقت ومعالجة الخلفية.

> **إجابات سريعة**  
> - **ما هي المكتبة الأفضل لتدوير الصور في Java؟** Aspose.PSD للـ Java.  
> - **هل يمكنني التدوير بأي درجة؟** نعم، طريقة `rotate` تقبل زاوية من نوع `float` (موجبة أو سالبة).  
> - **هل أحتاج إلى ترخيص للتطوير؟** نسخة تجريبية مجانية تكفي للاختبار؛ الترخيص مطلوب للإنتاج.  
> - **ما هي صيغ الصور المدعومة؟** PSD، JPEG، PNG، TIFF، GIF، BMP، والعديد غيرها.  
> - **كيف أضبط لون الخلفية للمساحات الفارغة؟** مرّر كائن `Color` إلى طريقة `rotate`.

## ما هو تدوير الصورة في Java؟

تدوير الصورة يعني تحويل مصفوفة البكسلات حول نقطة محورية (عادةً المركز) بزاوية معينة. في Java يمكنك تحقيق ذلك يدويًا باستخدام `Graphics2D`، لكن Aspose.PSD يختصر الرياضيات، يتعامل مع أعماق الألوان المختلفة، ويحافظ على معلومات الطبقات عند العمل مع ملفات PSD.

## لماذا تستخدم Aspose.PSD لتدوير الصور؟

- **الدقة:** تدوير بأي درجة كسرية دون فقدان الجودة.  
- **الأداء:** التخزين المؤقت المدمج (`image.cacheData()`) يسرّع التعامل مع الملفات الكبيرة.  
- **التحكم في الخلفية:** تحديد لون خلفية لملء الفجوات التي يخلقها التدوير.  
- **مرونة الصيغ:** تحميل PSD، وإخراج JPEG، PNG، أو أي صيغة مدعومة أخرى.

## المتطلبات المسبقة

قبل أن نبدأ، تأكد من وجود ما يلي:

1. **مجموعة تطوير جافا (JDK 8 أو أحدث)** – بيئة تطوير Java أو إعداد سطر أوامر يعمل.  
2. **Aspose.PSD للـ Java** – حمّل أحدث ملف JAR من [صفحة Aspose.PSD Java](https://reference.aspose.com/psd/java/).  
3. **ملف PSD تجريبي** – مثل `sample.psd` موجود في مجلد يمكنك الإشارة إليه من الكود.

## استيراد الحزم

أولًا، استورد الفئات التي سنحتاجها. هذه الاستيرادات تبقى كما هي بغض النظر عن زاوية التدوير التي تختارها.

```java
import com.aspose.psd.Color;
import com.aspose.psd.Image;
import com.aspose.psd.RasterImage;

import com.aspose.psd.imageoptions.JpegOptions;
```

## دليل خطوة بخطوة

### الخطوة 1: تعريف دليل المستند الخاص بك

حدد المجلد الذي يحتوي على ملف PSD المصدر ومكان كتابة الناتج.

```java
String dataDir = "Your Document Directory";
```

> **نصيحة محترف:** استخدم مسارًا مطلقًا أو `System.getProperty("user.dir")` لتجنب مفاجآت المسارات النسبية.

### الخطوة 2: تحديد مسارات ملفات المصدر والوجهة

```java
String sourceFile = dataDir + "sample.psd";
String destName = dataDir + "RotatingImageOnSpecificAngle_out.jpg";
```

يمكنك تغيير `destName` إلى أي امتداد مدعوم (`.png`، `.tiff`، إلخ) حسب احتياجاتك في الإخراج.

### الخطوة 3: تحميل الصورة

```java
RasterImage image = (RasterImage)Image.load(sourceFile);
```

`Image.load` يكتشف صيغة الملف تلقائيًا ويعيد كائن `RasterImage` ملموس للعمليات القائمة على البكسل.

### الخطوة 4: تخزين بيانات الصورة مؤقتًا (اختياري لكن موصى به)

```java
if (!image.isCached())
{
    image.cacheData();
}
```

التخزين المؤقت يحفظ بكسلات الصورة في الذاكرة، مما يسرّع التحولات اللاحقة — مفيد خاصةً للملفات الكبيرة من نوع PSD.

### الخطوة 5: تدوير الصورة

```java
image.rotate(20f, true, Color.getRed());
```

- **20f** – زاوية التدوير بالدرجات (float). غيّر هذه القيمة لتدوير بأي زاوية، مثل `-45f` للاتجاه العكسي.  
- **true** – الحفاظ على نسبة العرض إلى الارتفاع الأصلية مع توسيع القماش لاحتواء الصورة المدورة.  
- **Color.getRed()** – لون الخلفية الذي يملأ الزوايا الفارغة الناتجة عن التدوير. استبدله بـ `Color.getWhite()` أو أي لون مخصص حسب الحاجة.

### الخطوة 6: حفظ النتيجة

```java
image.save(destName, new JpegOptions());
```

`JpegOptions` يتيح لك التحكم في الجودة، الضغط، وإعدادات JPEG الأخرى. للحصول على إخراج غير مضغوط، استبدله بـ `PngOptions`.

## المشكلات الشائعة والحلول

| المشكلة | السبب | الحل |
|-------|-------|-----|
| **زوايا فارغة بعد التدوير** | لم يتم توفير لون خلفية | مرّر `Color` (مثل `Color.getWhite()`) إلى `rotate`. |
| **خطأ نفاد الذاكرة على ملفات PSD الكبيرة** | الصورة غير مخزنة مؤقتًا | استدعِ `image.cacheData()` قبل المعالجة. |
| **اتجاه الزاوية غير صحيح** | ارتباك بين القيم السالبة والموجبة | استخدم القيم السالبة للتدوير باتجاه عقارب الساعة (أو العكس حسب نظام الإحداثيات). |
| **التغييرات غير محفوظة** | نسيان استدعاء `save` | تأكد من تنفيذ `image.save(...)` بعد التدوير. |

## الأسئلة المتكررة

**س: هل يمكنني تدوير الصور ذات الشفافية باستخدام Aspose.PSD للـ Java؟**  
ج: نعم. المكتبة تحافظ على قناة ألفا؛ فقط تجنّب تحديد لون خلفية غير شفاف إذا أردت زوايا شفافة.

**س: هل هناك أي قيود على صيغ ملفات الصور المدعومة للتدوير؟**  
ج: لا. Aspose.PSD يدعم PSD، JPEG، PNG، TIFF، GIF، BMP، JPEG2000، WMF، EMF، والعديد غيرها.

**س: هل يمكنني تدوير الصور بزاوية سالبة؟**  
ج: بالتأكيد. مرّر قيمة `float` سالبة إلى `rotate` (مثال: `-30f`) للتدوير باتجاه عقارب الساعة.

**س: هل توفر Aspose.PSD معاينة صورة في الوقت الحقيقي أثناء التدوير؟**  
ج: الواجهة برمجية خادمة فقط. للحصول على معاينة حية، دمج الصورة المدورة في إطار UI (Swing، JavaFX) وتحديث العرض.

**س: هل هناك منتدى مجتمع لـ Aspose.PSD يمكنني طلب المساعدة فيه؟**  
ج: نعم، زر [منتدى Aspose.PSD](https://forum.aspose.com/c/psd/34) لطرح الأسئلة ومشاركة التجارب.
