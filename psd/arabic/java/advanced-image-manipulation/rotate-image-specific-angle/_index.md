# كيفية تدوير الصورة بزاوية محددة باستخدام Aspose.PSD للـ Java

## مقدمة

إذا كنت بحاجة إلى **كيفية تدوير الصورة** برمجياً في تطبيق Java، فإن Aspose.PSD للـ Java يقدم واجهة برمجة تطبيقات نظيفة وعالية الأداء تتولى عنك الجزء الأكبر من العملية. سواءً كنت تبني محرّراً للصور، أو تولّد صور مصغرة، أو تُعدّ أصولاً لخدمة ويب، فإن تدوير الصورة بدرجة دقيقة يُعدّ متطلباً شائعاً. في هذا الدرس سنستعرض العملية بالكامل—من تحميل ملف PSD إلى حفظ النتيجة المدورة—مع إبراز أفضل الممارسات مثل التخزين المؤقت ومعالجة الخلفية.

## إجابات سريعة
- **ما هي المكتبة الأفضل لتدوير الصور في Java؟** Aspose.PSD للـ Java.  
- **هل يمكنني التدوير بأي درجة؟** نعم، طريقة `rotate` تقبل زاوية من نوع `float` (موجبة أو سالبة).  
- **هل أحتاج إلى ترخيص للتطوير؟** النسخة التجريبية المجانية تكفي للاختبار؛ الترخيص مطلوب للإنتاج.  
- **ما صيغ الصور المدعومة؟** PSD، JPEG، PNG، TIFF، GIF، BMP، والعديد غيرها.  
- **كيف أضبط لون الخلفية للمساحات الفارغة؟** مرّر كائن `Color` إلى طريقة `rotate`.

## ما هو تدوير الصورة في Java؟

تدوير الصورة يعني تحويل مصفوفة البكسلات حول نقطة محورية (عادةً المركز) بزاوية معينة. في Java يمكنك تحقيق ذلك يدوياً باستخدام `Graphics2D`، لكن Aspose.PSD يختصر الحسابات، يتعامل مع أعماق الألوان المختلفة، ويحافظ على معلومات الطبقات عند العمل مع ملفات PSD.

## لماذا نستخدم Aspose.PSD لتدوير الصور؟

- **الدقة:** تدوير بأي درجة كسرية دون فقدان الجودة.  
- **الأداء:** التخزين المؤقت المدمج (`image.cacheData()`) يسرّع الملفات الكبيرة.  
- **التحكم في الخلفية:** تحديد لون الخلفية لملء الفجوات التي يخلقها التدوير.  
- **مرونة الصيغ:** تحميل PSD، وإخراج JPEG، PNG، أو أي صيغة مدعومة أخرى.

## المتطلبات المسبقة

قبل أن نبدأ، تأكد من وجود ما يلي:

1. **مجموعة تطوير Java (JDK 8 أو أحدث)** – بيئة تطوير Java أو إعداد سطر أوامر يعمل.  
2. **Aspose.PSD للـ Java** – حمّل أحدث ملف JAR من [صفحة Aspose.PSD Java](https://reference.aspose.com/psd/java/).  
3. **ملف PSD تجريبي** – مثال: `sample.psd` موجود في مجلد يمكنك الإشارة إليه من الكود.

## استيراد الحزم

أولاً، استورد الفئات التي سنحتاجها. هذه الاستيرادات تبقى كما هي بغض النظر عن زاوية التدوير التي تختارها.

```java
import com.aspose.psd.Color;
import com.aspose.psd.Image;
import com.aspose.psd.RasterImage;

import com.aspose.psd.imageoptions.JpegOptions;
```

## دليل خطوة بخطوة

### الخطوة 1: تعريف مسار دليل المستندات

حدد المجلد الذي يحتوي على ملف PSD المصدر ومكان كتابة الناتج.

```java
String dataDir = "Your Document Directory";
```

> **نصيحة احترافية:** استخدم مساراً مطلقاً أو `System.getProperty("user.dir")` لتجنب مفاجآت المسارات النسبية.

### الخطوة 2: تحديد مسارات الملفات المصدر والوجهة

```java
String sourceFile = dataDir + "sample.psd";
String destName = dataDir + "RotatingImageOnSpecificAngle_out.jpg";
```

يمكنك تغيير `destName` إلى أي امتداد مدعوم (`.png`، `.tiff`، إلخ) حسب احتياجاتك.

### الخطوة 3: تحميل الصورة

```java
RasterImage image = (RasterImage)Image.load(sourceFile);
```

`Image.load` يكتشف صيغة الملف تلقائياً ويعيد كائن `RasterImage` ملموس للعمليات القائمة على البكسل.

### الخطوة 4: تخزين بيانات الصورة مؤقتاً (اختياري لكن موصى به)

```java
if (!image.isCached())
{
    image.cacheData();
}
```

التخزين المؤقت يحفظ بكسلات الصورة في الذاكرة، ما يسرّع التحولات اللاحقة—مفيد خصوصاً للملفات الكبيرة من نوع PSD.

### الخطوة 5: تدوير الصورة

```java
image.rotate(20f, true, Color.getRed());
```

- **20f** – زاوية التدوير بالدرجات (float). غيّر هذه القيمة لتدوير بأي زاوية، مثل `-45f` للتدوير عكس اتجاه عقرب الساعة.  
- **true** – الحفاظ على نسبة الأبعاد الأصلية مع توسيع القماش لاحتواء الصورة المدورة.  
- **Color.getRed()** – لون الخلفية الذي يملأ الزوايا الفارغة الناتجة عن التدوير. استبدله بـ `Color.getWhite()` أو أي لون مخصص حسب الحاجة.

#### تدوير الصورة بالدرجات في Java

طريقة `rotate` تقبل أي قيمة عائمة، لذا يمكنك **تدوير الصورة بالدرجات** مثل `30f`، `90f`، أو `-15f`. القيم الموجبة تدور باتجاه عقرب الساعة، والقيم السالبة تدور عكسه.

#### تدوير الصورة بزاوية محددة باستخدام Aspose.PSD

بما أن الطريقة تتعامل مع `float`، يمكنك تحديد **زاوية تدوير صورة محددة** مثل `22.5f` للتحكم الدقيق في سير عمل التصميم الجرافيكي.

#### ملخص مثال تدوير الصورة في Java

جميع الخطوات أعلاه توضح سير عمل كامل لـ **rotate image java**—من تحميل الملف إلى حفظ النتيجة المدورة.

### الخطوة 6: حفظ النتيجة

```java
image.save(destName, new JpegOptions());
```

`JpegOptions` يتيح لك التحكم في الجودة، الضغط، وإعدادات JPEG الأخرى. للحصول على ناتج غير مضغوط، استبدله بـ `PngOptions`.

## المشكلات الشائعة والحلول

| المشكلة | السبب | الحل |
|--------|-------|------|
| **زوايا فارغة بعد التدوير** | عدم توفير لون خلفية | مرّر `Color` (مثال: `Color.getWhite()`) إلى `rotate`. |
| **خطأ نفاد الذاكرة على ملفات PSD الكبيرة** | عدم تخزين الصورة مؤقتاً | استدعِ `image.cacheData()` قبل المعالجة. |
| **اتجاه الزاوية غير صحيح** | خلط بين القيم السالبة والموجبة | استخدم القيم السالبة للتدوير عكس اتجاه عقرب الساعة (أو العكس حسب نظام الإحداثيات لديك). |
| **عدم حفظ التغييرات** | نسيان استدعاء `save` | تأكد من تنفيذ `image.save(...)` بعد التدوير. |

## الأسئلة المتكررة

**س: هل يمكنني تدوير الصور ذات الشفافية باستخدام Aspose.PSD للـ Java؟**  
ج: نعم. المكتبة تحافظ على قنوات ألفا؛ فقط تجنّب تحديد لون خلفية غير شفاف إذا أردت زوايا شفافة.

**س: هل هناك أي قيود على صيغ ملفات الصور المدعومة للتدوير؟**  
ج: لا. Aspose.PSD يدعم PSD، JPEG، PNG، TIFF، GIF، BMP، JPEG2000، WMF، EMF، والعديد غيرها.

**س: هل يمكنني تدوير الصور بزاوية سالبة؟**  
ج: بالتأكيد. مرّر قيمة سالبة إلى `rotate` (مثال: `-30f`) للتدوير عكس اتجاه عقرب الساعة.

**س: هل توفر Aspose.PSD معاينة صورة في الوقت الحقيقي أثناء التدوير؟**  
ج: الواجهة برمجية من جانب الخادم فقط. للحصول على معاينات حية، دمج البت ماب المدور في إطار UI (Swing، JavaFX) وتحديث العرض.

**س: هل هناك منتدى مجتمع لـ Aspose.PSD يمكنني طلب المساعدة فيه؟**  
ج: نعم، زر [منتدى Aspose.PSD](https://forum.aspose.com/c/psd/34) لطرح الأسئلة ومشاركة التجارب.
