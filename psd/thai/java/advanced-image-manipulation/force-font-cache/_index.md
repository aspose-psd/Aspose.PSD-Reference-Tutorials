# บันทึก PSD พร้อมฟอนต์และบังคับแคชฟอนต์ – Aspose.PSD Java

## คำแนะนำ

หากคุณต้องการ **บันทึก PSD พร้อมฟอนต์** อย่างรวดเร็วพร้อมให้แน่ใจว่ามีแบบอักษรที่ถูกต้องพร้อมใช้งาน คุณมาถูกที่แล้ว การแคชฟอนต์สามารถ **ปรับปรุงประสิทธิภาพภาพ** ได้อย่างมาก โดยเฉพาะเมื่อคุณประมวลผลไฟล์ Photoshop ขนาดใหญ่ซ้ำ ๆ ในบทเรียนนี้เราจะอธิบายขั้นตอนที่ต้องทำเพื่อบังคับแคชฟอนต์, **โหลดไฟล์ PSD** และสุดท้าย **บันทึก PSD พร้อมฟอนต์** ด้วย Aspose.PSD สำหรับ Java

## คำตอบสั้น ๆ
- **การบังคับแคชฟอนต์ทำอะไร?** ทำให้ Aspose.PSD สแกนฟอนต์ระบบใหม่เพื่อให้ฟอนต์ที่เพิ่งติดตั้งถูกจดจำทันที  
- **ควรรอเวลานานแค่ไหนสำหรับการติดตั้งฟอนต์?** ตัวอย่างโค้ดหยุดทำงานเป็นเวลา **2 นาที** ซึ่งโดยทั่วไปเพียงพอสำหรับการติดตั้งด้วยตนเอง  
- **สามารถใช้กับไฟล์ PSD ใดก็ได้หรือไม่?** ใช่ – ตราบใดที่ไฟล์เข้าถึงได้และฟอนต์ที่ต้องการได้ติดตั้งบนระบบ  
- **ต้องมีลิขสิทธิ์เพื่อรันโค้ดนี้หรือไม่?** จำเป็นต้องมีลิขสิทธิ์ชั่วคราวหรือเต็มสำหรับการใช้งานในผลิตภัณฑ์; เวอร์ชันทดลองฟรีใช้ได้สำหรับการประเมินผล  
- **รองรับเวอร์ชัน Java ใดบ้าง?** Aspose.PSD สำหรับ Java ทำงานกับ JDK 8 ขึ้นไป

## “บันทึก PSD พร้อมฟอนต์” คืออะไรและทำไมจึงสำคัญ?

การบันทึกไฟล์ PSD หลังจากแก้ไขเลเยอร์, ข้อความ หรือฟอนต์เป็นขั้นตอนสุดท้ายที่ทำให้การเปลี่ยนแปลงของคุณถาวร เมื่อแคชฟอนต์ไม่อัปเดต ไฟล์ที่บันทึกอาจใช้ฟอนต์เริ่มต้นแทน ทำให้เกิดความไม่สอดคล้องกันของภาพ การบังคับแคชก่อนจะทำให้การ **บันทึก PSD พร้อมฟอนต์** ฝังแบบอักษรที่ถูกต้องลงไป

## ทำไมต้องบังคับแคชฟอนต์ด้วย Aspose.PSD?

- **เพิ่มประสิทธิภาพ** – ลดความจำเป็นในการค้นหาฟอนต์ซ้ำ ๆ  
- **ความน่าเชื่อถือ** – รับประกันว่าไฟล์ PSD ที่บันทึกจะแสดงผลตรงตามที่ต้องการบนเครื่องใดก็ได้  
- **ความง่าย** – การเรียกเมธอดเดียว (`OpenTypeFontsCache.updateCache()`) ทำหน้าที่ทั้งหมด

## ข้อกำหนดเบื้องต้น

ก่อนเริ่มทำงาน ตรวจสอบให้แน่ใจว่าคุณมี:

- Java Development Kit (JDK) 8 หรือใหม่กว่า  
- ไลบรารี Aspose.PSD for Java (ดาวน์โหลดจาก [download link](https://releases.aspose.com/psd/java/))  
- ตัวอย่างไฟล์ PSD (`sample.psd`) ที่วางไว้ในโฟลเดอร์ที่คุณสามารถอ้างอิงจากโค้ดได้  

เมื่อทุกอย่างพร้อมแล้ว ไปสู่การทำงานกันเลย

## นำเข้าแพ็คเกจ

นำเข้าคลาสที่จำเป็นสำหรับทำงานกับภาพ PSD และแคชฟอนต์ วางบรรทัดเหล่านี้ไว้ด้านบนของคลาส Java ของคุณ:

```java
import com.aspose.psd.Image;
import com.aspose.psd.OpenTypeFontsCache;

import com.aspose.psd.fileformats.psd.PsdImage;
import java.io.Console;
import java.util.concurrent.TimeUnit;
```

## คู่มือขั้นตอนโดยละเอียด

### ขั้นตอนที่ 1: โหลดภาพ PSD (วิธีโหลด PSD image)

เราจะเริ่มด้วยการโหลดไฟล์ PSD ดั้งเดิม ซึ่งจะให้วัตถุภาพพื้นฐานที่เราจะบันทึกใหม่หลังจากอัปเดตแคชฟอนต์

```java
String dataDir = "Your Document Directory";

PsdImage image = (PsdImage)Image.load(dataDir + "sample.psd");
image.save(dataDir + "NoFont.psd");
```

- **คำอธิบาย:**  
  - `Image.load` อ่านไฟล์เข้าสู่หน่วยความจำ  
  - `image.save` สร้างสำเนาชื่อ **NoFont.psd** ที่สะท้อนสถานะ **ก่อน** ที่ฟอนต์ใหม่จะพร้อมใช้งาน  
  - ขั้นตอนนี้เป็นประโยชน์สำหรับการเปรียบเทียบและทำให้แน่ใจว่าการอัปเดตแคชจริง ๆ เปลี่ยนผลลัพธ์

### ขั้นตอนที่ 2: รอการติดตั้งฟอนต์ (java thread sleep – ปรับปรุงประสิทธิภาพภาพ)

ต่อไปเราจะให้ผู้ใช้มีเวลาติดตั้งฟอนต์ที่ต้องการด้วยตนเอง ในสถานการณ์จริงคุณอาจทำอัตโนมัติก็ได้ แต่การหยุดชั่วคราวนี้แสดงกลไกการรีเฟรชแคช

```java
System.out.println("You have 2 minutes to install the font");
Thread.sleep(2 * 60 * 1000);
OpenTypeFontsCache.updateCache();
```

- **คำอธิบาย:**  
  - `Thread.sleep` (เป็น **java thread sleep**) หยุดการทำงานเป็นเวลา **2 นาที** (2 × 60 × 1000 ms)  
  - `OpenTypeFontsCache.updateCache()` บังคับให้ Aspose.PSD สแกนฟอนต์ OpenType ของระบบใหม่ ทำให้ฟอนต์ที่เพิ่งติดตั้งพร้อมใช้งานทันทีสำหรับการเรนเดอร์  
  - การหยุดนี้ช่วย **ปรับปรุงประสิทธิภาพภาพ** โดยทำให้แคชสะท้อนฟอนต์ล่าสุดก่อนการโหลดครั้งต่อไป

### ขั้นตอนที่ 3: โหลด PSD ที่อัปเดตและ **บันทึก PSD พร้อมฟอนต์**

หลังจากรีเฟรชแคช เราจะโหลดไฟล์ PSD ดั้งเดิมอีกครั้ง ครั้งนี้ข้อมูลฟอนต์เป็นปัจจุบันและเราสามารถ **บันทึก PSD พร้อมฟอนต์** ได้อย่างถูกต้อง

```java
PsdImage image1 = (PsdImage)Image.load(dataDir + "sample.psd");
image1.save(dataDir + "HasFont.psd");
```

- **คำอธิบาย:**  
  - การโหลดครั้งที่สองจะดึงฟอนต์ที่เพิ่งติดตั้งมาใช้  
  - การบันทึกเป็น **HasFont.psd** จะสร้างไฟล์ที่มีการเรนเดอร์ฟอนต์ที่ถูกต้อง แสดงว่าการอัปเดตแคชทำงานสำเร็จ

## ปัญหาที่พบบ่อยและวิธีแก้

| ปัญหา | สาเหตุ | วิธีแก้ |
|-------|--------|----------|
| PSD ที่บันทึกยังแสดงฟอนต์เริ่มต้น | ฟอนต์ไม่ได้ติดตั้งในตำแหน่งที่ระบบสแกน | ติดตั้งฟอนต์ในโฟลเดอร์ฟอนต์ของระบบ (เช่น `C:\Windows\Fonts` บน Windows) แล้วเรียก `updateCache()` อีกครั้ง |
| `Thread.sleep` โยน `InterruptedException` | เมธอดไม่ได้จัดการกับข้อยกเว้นที่ตรวจสอบ | เพิ่ม `throws InterruptedException` ให้เมธอด `main` ของคุณ หรือห่อการเรียกในบล็อก try‑catch |
| `OpenTypeFontsCache.updateCache()` ไม่ทำงาน | รันบนเซิร์ฟเวอร์ headless ที่ไม่มีการตั้งค่าฟอนต์ | ตรวจสอบให้เซิร์ฟเวอร์มีฟอนต์ที่ต้องการติดตั้งและกระบวนการ Java มีสิทธิ์อ่าน |
| **จัดการฟอนต์ที่หายไป** – PSD แสดงด้วยฟอนต์สำรอง | ไฟล์ฟอนต์เสียหายหรือเข้าถึงไม่ได้ | ตรวจสอบความสมบูรณ์ของไฟล์ฟอนต์และให้แน่ใจว่ากระบวนการ Java มีสิทธิ์อ่าน |
